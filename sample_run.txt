Notes
-----
- preprocess, classifier, and router all completed successfully before the
  workflow paused on the [request_info] event, which is the bonus
  human-in-the-loop gate implemented in
  src/bug_triage/actions.py:NeedsHumanApprovalExecutor.
- The chosen route was "needs_human_approval_to_close" (a duplicate report,
  per src/bug_triage/router.py's rule 3).
- Running the same command with --auto-reject instead produces the same
  events up to [request_info], followed by:

      [output] needs_human_approval_to_close:

      Category: duplicate
      A human reviewer rejected the proposed action ('Close this report as
      duplicate.'); this report needs manual review.

- Without --auto-approve/--auto-reject, the CLI prompts interactively with
  "Approve this action? [y/n]: " at the [request_info] pause.
